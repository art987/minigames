## 1. 引言

### 1.1 编写目的

本用户手册旨在为"闪喵海报"移动端应用的用户提供详细的操作指南，帮助用户快速了解和使用本软件的各项功能。

### 1.2 软件概述

**闪喵海报**是一款面向中小商家、个体工商户的节日海报快速生成工具。软件采用Hybrid混合开发模式（uni-app x App壳 + H5业务页面），由原生应用容器负责底层资源调度，H5页面负责业务逻辑展示，通过JSBridge实现双向通信。

### 1.3 运行环境

- **操作系统**：Android 5.0+ / iOS 11.0+
- **网络要求**：需要互联网连接
- **权限需求**：相册权限（保存图片）

### 2.2 技术架构说明

本软件采用**Hybrid混合开发模式**，主要包含以下层次：

1. **表现层（App壳）**：使用uni-app x框架开发，负责WebView容器管理、原生能力调用、页面路由控制
2. **业务层（H5）**：使用HTML5+CSS3+JavaScript技术栈，负责海报编辑、模板选择、用户交互等核心业务
3. **支撑层（云服务）**：提供远程配置、数据存储、支付能力、图片CDN等服务

## 3. 功能说明

### 3.1 启动应用

#### 3.1.1 功能描述
用户点击手机桌面"闪喵海报"图标进入应用。应用启动后加载首页业务页面，并显示加载图、加载文案和进度提示。加载完成后进入海报模板与功能入口页面。

#### 3.1.2 技术实现
- App壳启动时请求远程配置（`webview-pages.remote.json`）
- 远程配置3秒内成功则覆盖本地配置；失败或超时时使用本地配置
- 加载首页H5页面（`https://peacelove.top/postdiy/index.html`）

### 3.5 保存海报到相册

#### 3.5.1 功能描述
用户点击保存按钮后，App将H5页面生成的海报图片保存到系统相册。保存成功后显示"已存入相册"提示，用户可选择"不再提示"。如果未授予相册权限，App会提示用户前往系统设置开启权限。

#### 3.5.2 保存流程

```javascript
// H5端生成海报并调用原生保存
async function handleDownload() {
    const canvas = await html2canvas(posterElement);
    const imageData = canvas.toDataURL('image/png');

    // 通过JSBridge调用原生保存
    if (window.postMessage) {
        window.postMessage({
            action: 'downloadImage',
            dataUrl: imageData,
            fileName: 'poster_' + Date.now() + '.png'
        });
    }
}
```

### 3.7 会员支付

#### 3.7.1 功能描述
用户在H5页面发起会员或订单支付时，App会识别支付链接并打开独立收银台页面。收银台页面支持跳转微信或支付宝完成支付。支付完成后返回App，App关闭收银台并回到原业务页面。

#### 3.7.2 支付流程
1. H5生成支付地址
2. App识别支付域名，打开独立收银台页（`pages/payment/index.uvue`）
3. 收银台加载支付页面
4. 用户选择微信/支付宝支付
5. App调用系统打开支付scheme（`weixin://`、`alipays://`）
6. 支付完成后回跳到业务站点
7. App关闭收银台，返回原业务页面

### 3.8 网络异常重试

#### 3.8.1 功能描述
当主业务页面因网络问题加载失败时，App展示自定义错误提示页面。用户检查网络后点击"重试一下"，App会重新加载当前页面。

#### 3.8.2 错误判定算法
- 读取失败事件中的URL
- 提取失败URL的host
- 提取当前业务页面host
- 两者一致时认为主业务页面失败，显示错误遮罩
- favicon或第三方资源失败不触发整页错误
