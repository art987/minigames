## 1. 引言

### 1.1 编写目的

本设计说明书详细描述"闪喵海报"App的软件架构、技术选型、核心算法和数据库设计，为软件开发、测试和维护提供技术参考。

### 1.2 项目背景

"闪喵海报"是一款面向中小商家的节日海报快速生成工具。商家无需设计基础，只需选择模板、填写信息，即可一键生成专业海报。

软件采用**Hybrid混合开发模式**（uni-app x App壳 + H5业务页面），通过JSBridge实现原生与H5的双向通信，兼具原生应用的流畅体验和Web应用的动态更新能力。

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| uni-app x | 基于UTS的跨平台开发框架，性能接近原生 |
| UTS | Uni Type Script，uni-app x的底层语言 |
| JSBridge | JavaScript与原生代码的桥接层 |
| WebView | 原生应用中用于加载网页的组件 |
| H5 | HTML5技术栈（HTML+CSS+JavaScript） |
| .uvue | uni-app x的页面文件格式 |
| .uts | uni-app x的脚本文件格式 |

## 3. 核心模块设计

### 3.1 App壳模块

#### 3.1.1 WebView宿主模块

**所在文件**：`components/bridge-webview/bridge-webview.uvue`

**功能说明**：
1. 根据配置加载对应H5页面
2. 控制顶部标题栏、返回按钮、刷新按钮和底部导航栏
3. 监听WebView的加载、成功、失败、下载、消息事件
4. 注入JSBridge，增强H5页面与App的通信能力
5. 在网页异常时展示App侧自定义错误遮罩

#### 3.1.2 远程配置模块

**所在文件**：`config/webview-pages.uts`

**功能说明**：
1. 本地保存默认页面地址、tab配置、加载样式、错误遮罩、更新配置
2. 启动后请求远程配置 `https://peacelove.top/postdiy/webview-pages.remote.json`
3. 远程配置3秒内成功则覆盖本地配置；失败、超时或格式错误时使用本地配置
4. 支持通过 `showInTabBar` 控制页面是否显示在底部导航中
5. 支持通过 `secureWindowEnabled` 控制Android防截图和防录屏

#### 3.1.3 JSBridge交互模块

**所在文件**：`components/bridge-webview/bridge-webview.uvue`

**主要消息类型**：

| action | 说明 | 参数 |
|--------|------|------|
| syncTitle | 同步页面标题和URL | title, url |
| syncWebState | 同步cookie和localStorage | cookie, localStorageJson |
| copyText | 复制文本到剪贴板 | text |
| openTaobao | 唤起淘宝App | url |
| openPaymentUrl | 打开支付收银台 | url |
| downloadImage | 保存图片到相册 | dataUrl, fileName |

#### 3.1.4 支付收银台模块

**所在文件**：`pages/payment/index.uvue`

**功能说明**：
1. App壳识别支付域名后，打开独立收银台页面
2. 收银台页面加载支付地址，避免主业务WebView被支付流程污染
3. 对 `weixin://`、`alipays://` 等支付scheme调用系统打开
4. 支付成功回跳到业务站点时，关闭收银台并返回原业务页面

### 3.2 H5业务模块

#### 3.2.1 首页模块

## 4. 核心算法设计

### 4.1 远程配置回退算法

**算法目标**：保证App在远程配置不可用时仍可启动和使用

**处理流程**：
1. 启动时先生成本地默认配置
2. 请求远程JSON配置文件
3. 设置3秒超时保护
4. 远程数据为空、解析失败、字段缺失或请求失败时使用本地配置
5. 远程数据有效时，仅覆盖配置中存在且合法的字段
6. 最终将可用配置回调给页面应用

### 4.2 WebView错误判定算法

**算法目标**：区分主页面加载失败和第三方资源加载失败

**处理流程**：
1. 读取失败事件中的 `src` 和 `fullUrl`
2. 忽略favicon请求失败
3. 提取失败URL的host
4. 提取当前业务页面host
5. 两者一致时认为主业务页面失败，显示错误遮罩
6. 两者不一致时认为是第三方资源失败，仅记录日志，不影响用户继续使用

### 4.3 支付会话防重复算法

**算法目标**：避免同一订单重复拉起多个收银台

**处理流程**：
1. 从支付URL中提取 `O_id`、`redirect_url`、`out_trade_no` 或 `prepay_id`
2. 生成支付会话key
3. 如果本地已有未过期支付锁，比较当前key与锁定key
4. 同一支付会话直接忽略新跳转
5. 新支付会话会替换旧支付锁
6. 支付完成、取消或超时后清理支付锁

### 4.4 图片保存算法

**算法目标**：兼容H5生成海报的多种图片格式

**处理流程**：
1. 如果是base64或data URL，解析图片类型和base64内容
2. 写入App用户数据目录生成临时图片文件
3. 如果是远程URL，先调用下载接口得到临时文件
4. 调用系统相册保存接口
5. 根据保存结果显示成功提示、失败提示或权限引导

### 4.5 节日自动推荐算法（H5）

**算法目标**：根据当前日期推荐相关节日

**代码实现**：

```javascript
function autoSelectByDate() {
    const today = new Date();
    today.setHours(0, 0, 0, 0);

    const currentYear = today.getFullYear();
    const festivalDates = window.festivalDates[currentYear] || {};

    let selectedFestival = null;
    let minDays = Infinity;

    for (const [name, dateStr] of Object.entries(festivalDates)) {
        const festivalDate = new Date(dateStr.split(' ')[0]);
        const daysUntil = Math.ceil((festivalDate - today) / (1000 * 3600 * 24));

        if (daysUntil >= 0 && daysUntil < minDays) {
            minDays = daysUntil;
            selectedFestival = name;
        }
    }

    return {
        month: today.getMonth() + 1,
        festival: selectedFestival,
        daysUntil: minDays
    };
}
```

## 6. 接口设计

### 6.1 远程配置接口

**接口地址**：`https://peacelove.top/postdiy/webview-pages.remote.json`

**请求方式**：GET

**主要返回内容**：
- `secureWindowEnabled`：是否开启防截图
- `appUpdate`：应用更新配置
- `tabBarItems`：底部导航和页面地址配置
- `tabBarStyle`：底部导航样式配置
- `webviewLoadingStyle`：页面加载样式配置
- `webviewErrorOverlayStyle`：错误遮罩样式配置

### 6.2 JSBridge消息接口

**接口方式**：H5页面向App WebView宿主发送消息

**主要字段**：
- `action`：动作类型
- `title`：页面标题
- `url`：目标链接或图片地址
- `dataUrl`：base64图片内容
- `fileName`：保存文件名
- `text`：复制文本
- `host`：H5当前域名
- `cookie`：H5 cookie
- `localStorageJson`：H5 localStorage内容

### 6.3 支付跳转接口

**接口方式**：H5生成支付地址后，由App识别并打开独立收银台页面

**支持域名**：
- `zpayz.cn`
- `www.zpayz.cn`
- `wx.tenpay.com`
- `payapp.weixin.qq.com`
- `mall.z-pay.cn`

**支持scheme**：
- `weixin://`
- `alipays://`
