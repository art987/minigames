## 四、避坑指南

### ✅ 必须做的：

1. **源代码必须包含App壳代码**
   - 不能只交H5代码
   - 否则会被判定为"网页设计"而非"计算机软件"

2. **截图必须是APP环境**
   - 顶部要有状态栏（电量、时间）
   - 不能是浏览器打开网页的截图

3. **文档强调混合开发**
   - 说明"原生容器负责底层调度，H5负责业务展示"
   - 重点描述JSBridge交互

4. **保留业务变量名**
   - 不要混淆成 `a`、`b`、`c`
   - 保留如 `handleUserLogin()`、`autoSelectByDate()` 等

### ❌ 避免的：

1. **只交H5代码**：直接驳回
2. **代码全是第三方API**：要有自己的业务逻辑
3. **截图没有状态栏**：无法证明是APP
4. **去除所有注释**：缺乏独创性
